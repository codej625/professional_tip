# Hyper-V

<br />
<br />

* Hyper-V 란?

---

```
Windows에 내장된 가상화 프로그램이다.

한 대의 컴퓨터에서 여러 개의 가상 컴퓨터 (VM)을 돌릴 수 있게 해준다.
```

<br />
<br />
<br />
<br />

1. 가상 스위치란?

```
실제 네트워크 스위치처럼,
가상 컴퓨터들을 연결해주는 네트워크 허브 역할을 한다.
```

<br />

| 종류            | 인터넷 연결 | 외부 → VM 접근 | 호스트 ↔ VM 통신 | VM ↔ VM 통신 | 주요 용도 / 비유                                                                                    |                                                      
|----------------|----------|--------------|---------------|-------------|--------------------------------------------------------------------------------------------------|
| External (외부) | O        | O            | O             | O           | **공개 웹 서버**처럼 외부에서 직접 접근해야 하는 VM용. <br> (비유: 인터넷에 직접 연결된 컴퓨터)                     |
| Internal (내부) | X        | X            | O             | O           | **App/DB 서버**처럼 내부망에서만 통신하고 호스트 PC가 관리해야 할 때. <br> (비유: 집 안에서만 쓰는 공유기/WiFi)      |
| Private (사설)  | X        | X            | X             | O           | **완전히 격리된 테스트 환경** 등 VM들끼리만 통신하고 외부/호스트와는 완벽히 차단할 때. <br> (비유: 그들만의 비밀 네트워크) |

<br />
<br />
<br />

2. 3-tier 구조

```
간단히 말하면,
웹 서비스를 3개 층으로 나누어서 만드는 구조를 말한다.
```

<br />

`1) Presentation Tier - (프레젠테이션 계층)`

```
역할 : 사용자와 직접 만나는 부분
예시 : 웹 브라우저, 모바일 앱 화면
백엔드 적용 시 : Nginx (웹서버 + 로드밸런서)
```

<br />

`2) Application Tier - (애플리케이션 계층)`

```
역할: 비즈니스 로직, 데이터 처리
예시: 로그인 처리, 계산, API 서버
우리 프로젝트: App Server (백엔드 서버)
```

<br />

`3) Data Tier - (데이터 계층)`

```
역할: 데이터 저장, 관리
예시: MySQL, PostgreSQL, MongoDB
우리 프로젝트: Database Server
```

<br />
<br />

`동작 흐름`

```
인터넷 (외부)
    ↓
[External Switch] ← Nginx VM 연결
    ↓
[Internal Switch] ← Nginx VM + App Server VM 연결
    ↓
[Private Switch] ← App Server VM + Database VM 연결
```
