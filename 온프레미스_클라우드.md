# 온프레미스와 클라우드의 서버 구성 차이

<br />
<br />

* 물리서버 vs 클라우드

---

```
서버를 구축하다 보면,
온프레미스 환경 혹은 클라우드 환경과 같은 여러 가지 환경에서
서버를 구축하게 된다.

이때 쉽게 비교할 만한 자료를 만들어보았다.
```

1. 네트워크 구성 요소 비교

| 구성 요소 | 온프레미스 | 클라우드 (AWS) | 설명 |
|---------|----------|--------------|------|
| **게이트웨이** | 공유기 (하드웨어) | Internet Gateway (IGW) | 외부 인터넷과 내부 네트워크 연결 |
| **네트워크 공간** | 공유기 내부 네트워크 (192.168.0.0/24) | VPC (10.0.0.0/16 등) | 논리적으로 격리된 네트워크 공간 |
| **서브넷** | 단일 서브넷 (보통 /24) | 여러 서브넷 (Public/Private 등) | 네트워크 세분화 |
| **IP 대역** | 사설 IP (192.168.0.0/24) | 사설 IP (10.0.0.0/8 등) | 내부 네트워크 IP |
| **사용 가능 IP 수** | 254개 (/24 기준) | 수만~수십만 개 (/16 기준) | 서브넷 크기에 따라 다름 |
| **로드밸런서** | nginx VM (수동 구성) | ELB/ALB/NLB (관리형 서비스) | 트래픽 분산 |
| **가상 머신** | Hyper-V VM | EC2 Instance | 실제 워크로드 실행 환경 |
| **리소스 관리** | 물리 서버 직접 관리 | 클라우드 콘솔/API | 리소스 생성 및 관리 방식 |
| **확장성** | 하드웨어 제약 | 거의 무제한 | 리소스 추가 용이성 |
| **고가용성** | 수동 구성 필요 | 자동 제공 (여러 AZ) | 장애 대응 능력 |
| **비용** | 초기 투자 + 유지보수 | 사용한 만큼 지불 | 비용 구조 |
| **백업** | 수동 설정 필요 | 자동 스냅샷/백업 | 데이터 보호 |
| **모니터링** | 직접 구축 필요 | CloudWatch 등 제공 | 리소스 모니터링 |

2. 네트워크 흐름 비교

| 단계 | 온프레미스 | 클라우드 |
|-----|----------|---------|
| **1. 인터넷 진입** | 공인 IP (ISP 할당) | 공인 IP (Elastic IP) |
| **2. 게이트웨이** | 공유기 (NAT) | Internet Gateway |
| **3. 네트워크** | 내부 네트워크 (192.168.0.0/24) | VPC (10.0.0.0/16) |
| **4. 서브넷** | 단일 서브넷 | Public/Private Subnet |
| **5. 로드밸런서** | nginx VM | ELB |
| **6. 백엔드** | 쿠버네티스 클러스터 (VM) | EC2 인스턴스 |

3. 주요 차이점

| 특징 | 온프레미스 | 클라우드 |
|-----|----------|---------|
| **설정 복잡도** | 높음 (수동 구성) | 낮음 (자동화) |
| **유연성** | 제한적 (하드웨어 의존) | 높음 (즉시 확장/축소) |
| **관리 부담** | 높음 (직접 관리) | 낮음 (관리형 서비스) |
| **초기 비용** | 높음 (하드웨어 구매) | 낮음 (사용한 만큼) |
| **장기 비용** | 예측 가능 | 사용량에 따라 변동 |
| **보안** | 직접 관리 | 클라우드 제공 보안 기능 |
| **재해 복구** | 수동 구성 | 자동 백업/복구 |
