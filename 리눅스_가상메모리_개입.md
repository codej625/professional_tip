# 리눅스의 가상메모리

<br />
<br />

* 가상메모리?

---

```
가상 메모리는 RAM 부족을 해결하기 위해 HDD, SSD 등의 보조기억장치를 주기억장치처럼 활용하는 기술이다.

성능 향상보다는 시스템을 안전하고 효율적으로 구동시키기 위해 꼭 필요한 안전장치이다.
```

<br />
<br />
<br />
<br />

1. 가상메모리가 설정되어 있는지 확인하기

```
# 가상 메모리 용량을 바로 확인
free -h

# 상세 보기
swapon --show
```

<br />
<br />
<br />

2. 가상 메모리 만들기

```
# 4GB 크기의 빈 파일 생성 (원하는 용량에 따라 4G, 8G 등으로 변경 가능)
sudo fallocate -l 4G /swapfile

# 파일 권한을 루트(보안) 권한으로만 제한
sudo chmod 600 /swapfile

# 이 파일을 스왑 공간으로 포맷
sudo mkswap /swapfile

# 스왑 파일 활성화
sudo swapon /swapfile

# 부팅 시 자동 적용하기 (재부팅하면 사라지므로, 영구 적용을 위해 설정 파일에 등록해야 해야 함)
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

<br />
<br />
<br />

3. 가상 메모리 확장하는 방법

```
# 기존 가상 메모리 활성화 해제
sudo swapoff /swapfile

# 원하는 새 용량(예시 -> 8GB)으로 파일 크기 재할당
sudo fallocate -l 8G /swapfile

# 스왑 공간으로 다시 포맷
sudo mkswap /swapfile

# 가상 메모리 다시 활성화
sudo swapon /swapfile
```

4. swappiness

```
RAM이 부족해지기 시작할 때,
가상 메모리(SSD/HDD)를 얼마나 자주,
빨리 쓸 것인가? 를 결정하는 비율 설정이다.

우분투의 기본값은 60으로 되어있다.

예시 -> 0에 가까우면 최대한 RAM에 데이터를 유지하고, RAM이 꽉 차서 터지기 직전까지 가상 메모리를 사용 X
```

<br />

```
# 현재 설정 확인
cat /proc/sys/vm/swappiness # 기본값 60

# 서버 환경 권장값으로 변경 (재부팅 시 초기화)
sudo sysctl vm.swappiness=10

# 영구 적용
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
```
