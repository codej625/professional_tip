* 모듈?
---

```
시스템 전체를 구성하는 독립적인 기능 단위를 의미한다.

더 큰 시스템의 일부로서 특정 역할이나 기능을 캡슐화(감싸서) 제공한다.

논리적인 분할 단위가 될 수도 있고,
실제로 배포되는 단위가 될 수도 있다.

대규모 프로젝트를 개발할 때 코드를 구조화하고 관리하기 위해 사용된다.

각 모듈은 자체적인 책임과 역할을 가진다.

빌드 시스템(Gradle 등)의 관점에서는,
하나의 프로젝트를 여러 개의 작은 하위 프로젝트(모듈)로 나누어 관리할 때 이 각각을 모듈이라고 부르기도 한다.

이 경우 한 모듈이 다른 모듈에 의존할 수 있다.
```

1. 모듈의 구성

```
모듈을 라이브러리라고 생각하는 사람도 있고,
어떤 코드의 집합 혹은 단위라고 생각하는 사람들도 있다.

모듈은 쉽게 생각하면,
레고로 만든 자동차라고 할 때, 엔진 부분, 바퀴 부분, 차체 부분처럼 기능별로 나눈 각각의 서브 파트이다.

각 서브 파트(모듈)는 여러 개의 레고 블록(라이브러리나 직접 만든 코드)으로 구성될 수 있으며,
이 서브 파트들이 합쳐져서 전체 자동차(시스템)가 된다.

모듈은 시스템을 구성하는 논리적 또는 물리적인 기능 단위 (프로젝트 내부를 나눈 것일 수도 있다) 이거나,
하나의 큰 프로젝트를 여러 개의 하위 모듈(서브 프로젝트)로 나누어 관리하는 멀티 모듈 일수도 있는것이다.
```

```
모듈은 파이썬, 자바스크립트(노드)에서 많이 사용되는 단어이고
헷갈리는 사람들이 많기 때문에 정리해 봤다.
```
