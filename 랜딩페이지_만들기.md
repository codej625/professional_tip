# 랜딩 페이지

<br />
<br />

* 랜딩 페이지란?

---

```
랜딩 페이지(Landing Page)는
사용자가 특정 링크를 클릭하거나 광고를 통해 처음 도착하는 웹 페이지이다.

한 페이지에서 특정 목표 달성에 집중하고,
명확한 행동을 유도한다.
(회원가입, 구매, 다운로드 등)
```

<br />
<br />
<br />
<br />

1. 랜딩페이지를 만들 때 알아두면 좋은 것들

```
랜딩페이지는 일반적으로 크게 기술적인 기능을 포함하지는 않기 때문에,
굉장히 간단한 화면으로 오해하기 십상이다.

하지만 실제로 가장 거액을 들여 광고하는 페이지인 만큼
여러 가지 측면에서 세심한 작업이 필요하다.
```

<br />

`단일 목표 설정`

```
한 페이지에 하나의 명확한 목표를 세워야 한다.

랜딩페이지에서 동시에 여러 가지 성과를 거두려고 한다면,
오히려 아무 성과도 거두지 못할 가능성이 크다.
```

<br />

`사용자 경험`

```
로딩 속도, 모바일 최적화 반응형 디자인, 직관적 레이아웃, 핵심 정보는 첫 화면 표시 등
사용자 경험이 굉장히 중요하다.

특히 요즘같이 갤럭시 폴드, 플립같은 가로가 좁은 화면에서도 이질감이 없는지
꼼꼼하게 테스트해야 한다.
```

<br />

`신뢰성 구축`

```
명확한 정보와 개인정보 보호 정책 링크,
보안 인증 표시 (SSL, 결제 보안 등),
거짓 없는 실제 고객 사례로 최대한 허위 정보를 담지 않는 게 결국 전환으로 이어진다.
```

<br />

`기술적 최적화 (검색엔진 최적화)`

```
SEO, 접근성 (키보드 네비게이션, 스크린 리더 지원)
크로스 브라우저 테스트, Google Analytics, Google Search Console,
네이버 서치어드바이저 등을 설치하여 검색엔진 최적화를 한다.
```

<br />

`지속적 개선`

```
가장 중요한 건 지속적인 개선 작업이다.

사용자의 피드백을 바탕으로 전환율 추적 및 개선작업이 이뤄져야 한다.
```

<br />

`법적 준수`

```
개인정보 처리방침을 명시하고,
이용약관 제공, 광고 규정 준수 (특히 건강/금융 분야) 등
전문가의 검증을 통한 법적 준수가 필요하다.
```

2. 반드시 피해아 할 것들

```
랜딩페이지에서는 너무 많은 선택지 제공하거나,
복잡한 폼 (최소한의 필수 정보만),
산만한 디자인, 느린 로딩 속도, 모바일 미지원은 반드시 피해야 할 부분이다.
(반응형이 아닌 경우 모바일 페이지를 별도로 만들어 반드시 모바일 대응이 필요하다.)
```
